# Projekt zespołowy – aplikacja webowa (projekt symulacyjny)

## Opis projektu
Repozytorium zawiera dokumentację projektu symulacyjnego, którego celem było przećwiczenie
koordynacji pracy zespołu przy realizacji aplikacji webowej.
Projekt skupiał się na planowaniu zadań, organizacji pracy, harmonogramowaniu oraz monitorowaniu postępów.

Rola w projekcie: Koordynator projektu / Asystent Kierownika Projektu

## Zakres projektu
- Aplikacja webowa (część frontendowa)
- Projekt zespołowy
- Dynamiczny zakres z możliwością wprowadzania zmian w trakcie realizacji

## Zakres odpowiedzialności
- Analiza potrzeb i zebranie wymagań projektowych
- Określenie zakresu funkcjonalnego aplikacji
- Utworzenie oraz priorytetyzacja backlogu zadań
- Zaplanowanie harmonogramu prac
- Rozdzielanie zadań pomiędzy członków zespołu
- Monitorowanie postępu realizacji zadań
- Aktualizacja statusów prac i reagowanie na opóźnienia
- Koordynacja komunikacji w zespole
- Dostosowywanie planu projektu do zmian zakresu

## Wykorzystywane narzędzia
- Jira / Trello – zarządzanie zadaniami i backlogiem
- Google Docs – dokumentacja projektowa
- GitHub – repozytorium dokumentacji projektu

## Podział zadań i harmonogram

| Rola | Zadanie | Opis | Szacowany czas |
|------|--------|------|----------------|
| Koordynator projektu | Analiza wymagań | Zebranie i opis wymagań funkcjonalnych | 6 h |
| Koordynator projektu | Backlog zadań | Utworzenie i priorytetyzacja zadań | 4 h |
| Frontend Developer | Implementacja UI | Układ strony, responsywność | 16 h |
| Frontend Developer | Logika aplikacji | Formularze i interakcje | 12 h |
| Content / Grafika | Przygotowanie treści | Teksty i materiały graficzne | 8 h |
| Tester | Testy funkcjonalne | Weryfikacja poprawności działania | 6 h |

## Rezultat projektu
Efektem projektu była kompletna dokumentacja obejmująca zakres funkcjonalny, backlog zadań,
harmonogram prac oraz proces monitorowania postępów.
Projekt pozwolił na praktyczne przećwiczenie organizacji pracy zespołu oraz zrozumienie cyklu życia projektu IT.
